Remarks

The operator property determines the condition type. Operators map to conditions as follows:

  • begins_with - QueryConditionBeginsWith
  • between - QueryConditionBetween
  • <, <=, =, >, >=, <> - QueryConditionComparison
  • contains - QueryConditionContains
  • attribute_exists, attribute_not_exists - QueryConditionExists
  • in - QueryConditionIn
  • and, or - QueryConditionGroup
  • not - QueryConditionNot

Note that the and, or, and not operators permit nested conditions.

For more info, see the DynamoDB filter expression documentation.
